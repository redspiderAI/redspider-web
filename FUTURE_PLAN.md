# 项目未来规划（FUTURE PLAN）

本文档为本项目（当前版本 v0.0.1）的后续工作与实施路线图。目标是把当前已完成的首页初步设计扩展为一个可上线、性能良好且易维护的网站，并建立图像资源与联系方式的标准化流程。

---

## 一、当前状态（Summary）

- 已完成：首页（初步设计、静态实现）
  - 主要文件：`src/App.tsx`, `src/main.tsx`, `src/components/*`
- 未完成：其他界面设计、图片导入、图片清晰度不足、Logo 添加、联系方式（公司地址/电话/公司资料）添加、首页组件动画和字体优化。

---

## 二、目标（Goals）

1. 完成所有页面的界面设计与实现（`关于我们`、`核心业务`、`项目案例`、`人才发展`、`联系我们` 等）。
2. 为图片提供可靠的导入与处理流水线，保证展示质量并兼顾性能。
3. 添加并统一公司 Logo 与站点元信息（favicon、社媒预览图）。
4. 补充联系方式与公司资料，放置可操作的交互（点击拨号/打开地图/复制）。
5. 优化组件动画（性能与无障碍），确保支持 prefers-reduced-motion。
6. 建立基本测试、CI/CD 与部署流程，保证可持续交付。

---

## 三、未完成项的拆解与技术建议

1) 其他界面设计（高优先）
   - 任务：为 `关于我们`、`核心业务`、`项目案例`、`人才发展`、`联系我们` 完成视觉稿与响应式实现。
   - 输出：每页的组件清单、移动/平板/桌面断点样式、必要的占位与文案。
   - 验收标准：页面在常见视口（375/768/1366 px）下无布局溢出，样式与首页风格一致。

2) 图片导入（中高优先）
   - 方案：实现一个本地资源管理流程（先在 `data/pic/` 中统一管理），并在未来考虑接入媒体服务（Cloudinary/Imgix）或 CDN。
   - 技术建议：前端使用 `<picture>` + `srcset`，构建时通过 `vite-imagetools` 或 Node + `sharp` 生成多分辨率与 WebP/AVIF 格式。
   - 功能点：支持拖放上传（开发环境）、图片元数据（title、alt、来源）、预览与替换。

3) 图片清晰度不足（中优先）
   - 先行策略：优先替换为更高分辨率原图；若原图不可得，可选方案：
     - 在构建时使用 `sharp` 做高质量重采样与去噪；
     - 对极低分辨率图，考虑使用 AI 放大（如 Real-ESRGAN）作为可选离线步骤（复杂、需预处理）。
     - 使用ai高清网站，如[waifu2x](https://www.waifu2x.net/?lang=en),效果不好（
   - 验收标准：关键展示图片在 2x 设备下至少看起来清晰且无明显马赛克。

4) Logo 添加（高优先）
   - 要求：提供矢量格式（SVG）为首选；生成 favicon（svg/png）和站点预览图（og:image）。
   - 集成点：`src/components/Navigation.tsx`（头部），`index.html`（favicon / meta）。

5) 联系方式与公司资料（高优先）
   - 任务：在 `src/components/Contact.tsx` 中补充字段：公司地址（带地图链接）、客服电话（tel:）、公司简介、可下载/查看的公司资料（PDF）。
   - 数据建议：将联系方式放入 `src/data/contact.json` 并由 `Contact` 组件读取，便于后续维护与 i18n。

6) 首页组件动画优化（中优先）
   - 建议：
     - 使用 transform/opacity 以避免布局重排；
     - 使用 IntersectionObserver 做“进入视口触发”，并用 CSS 变量控制延迟与错列；
     - 支持 `prefers-reduced-motion`；
     - 减少动画持续时间与同时运行动画数量，避免影响首屏渲染。
   - 验收标准：Lighthouse 动画/性能得分无明显回退；在弱设备上动画平滑且可关闭。

7) 其他（中低优先）
   - 响应式增强、无障碍（ARIA/语义化）、SEO（meta、og）、性能优化（Lighthouse >= 90 优先目标）。

## 四、技术契约（Contract）

- 输入：设计稿、图片资源、公司资料文本。
- 输出：可部署的网站（静态构建），文档（本文件 + README 更新）、测试套件（基础）。
- 数据结构示例（`src/data/contact.json`）：

```json
{
  "company": "公司名称",
  "address": "地址文本",
  "phone": "+86-10-12345678",
  "email": "contact@example.com",
  "profileUrl": "/assets/company-profile.pdf"
}
```

- 主要错误模式：图片缺失、超大图片导致构建时间长、动画影响首屏渲染。

## 五、里程碑与估时（建议）

- 短期（1 周）
  1. 设计本规划文档
  2. 开始 `About` 页面视觉与响应式实现（2 天）
  3. 在 `data/pic` 中整理现有图片并补充元数据（1 天）

  2. 实现图片导入/优化流水线（vite-imagetools + sharp），生成多分辨率资源

## 六、验收标准（Definition of Done）

- 各页面在三种断点下无布局溢出，视觉风格统一。
- 关键图片在高分辨率屏幕下清晰，加载采用延迟加载与占位图策略。
- 站点包含 Logo、favicon 与基本 meta（title/description/og）。
- Contact 页面展示并可操作公司地址与电话信息。
- 动画在支持设备上流畅，在 `prefers-reduced-motion` 下关闭。
- CI 包含 lint、build、test 并通过自动部署。

---

## 七、风险与缓解

- 风险：高质量原始图片不可得 → 缓解：联系素材提供方或使用 AI/离线放大，但需人工验收。
- 风险：图片处理耗时 → 缓解：CI 中分离资源处理步骤，构建时使用缓存。
- 风险：动画对性能的影响 → 缓解：使用 transform/opacity、减少并行动画、提供开关。
